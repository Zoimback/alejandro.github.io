CI/CD Pipeline

DevOps Engineer

Descripción del proyecto

Este pipeline está diseñado para generar una imagen Docker, compilar el código, realizar pruebas, verificar la calidad del código y subir los artefactos generados para su despliegue. Puedes encontrar el código en este enlace -> Auto-piops

Funciones

  • Generación de imagen Docker: Creación de una imagen basada en el archivo Dockerfile y su respectivo contenedor.
  • Compilación: Compilación del código fuente.
  • Pruebas: Ejecución de pruebas automatizadas para garantizar la estabilidad y funcionalidad del código.
  • Calidad del código: Análisis de la calidad del código para asegurar el cumplimiento de estándares.
  • Subida de artefactos: Posibilidad de subida de los artefactos a una aplicación (Ej. Nexus SonarType).
  • Limpieza: Eliminación del contenedor, imagen y workset creado al finalizar el proceso.

Instrucciones de Instalación

Necesitas tener Jenkins instalado. Una vez instalado necesitarás configurarlo para que apunte a este repositorio. Se trata de un scripted pipeline por lo que habrá que configurarlo como shared-libraries.

Casos de Uso

Para esta pipeline, se ha creado esta template. Una vez creado el repositorio con la template, habrá que modificar del Dockerfile y el .ci-config.yml en función del proyecto (se encunetran dentro de la carpeta config). No modificar el Jenkinsfile.

Arquitectura

Se ha diseñado la pipeline para seguir el orden de la imagen.
